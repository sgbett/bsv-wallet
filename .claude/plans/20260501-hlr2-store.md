# Plan: HLR #2 — Store Implementation

## Context

HLR #1 delivered the schema and Sequel models (Layer 2b). The Store is the Layer 2a component that orchestrates those models into the phase-based action lifecycle. It's the largest interface (20 methods) and the backbone everything else calls through.

The Store receives and returns plain hashes — no ORM objects leak through the interface boundary. It wraps Sequel model operations in database transactions where atomicity is needed. It contains no BRC-100 business logic — that's the engine's job (Layer 3).

**Source issue:** sgbett/bsv-wallet#2

### Method-by-method implementation

#### Action Lifecycle

**`create_action(action:, inputs: [])`** — Phase 1: Lock

```ruby
def create_action(action:, inputs: [])
  db.transaction do
    record = Action.create(
      description:  action[:description],
      broadcast:    action[:broadcast]&.to_s || 'delayed',
      nlocktime:    action[:nlocktime] || 0,
      version:      action[:version],
      outgoing:     action.fetch(:outgoing, true),
      satoshis:     action[:satoshis],
      input_beef:   action[:input_beef]
    )

    locked = []
    inputs.each do |inp|
      # ON CONFLICT DO NOTHING — returns nil if output already claimed
      result = db[:inputs].insert_conflict(target: :output_id).insert(
        action_id:  record.id,
        output_id:  inp[:output_id],
        vin:        inp[:vin],
        nsequence:  inp[:nsequence] || 4_294_967_295,
        description: inp[:description]
      )
      locked << inp[:output_id] if result
    end

    if inputs.any? && locked.size < inputs.size
      raise db.rollback  # rollback, let caller retry with different outputs
    end

    action_to_hash(record)
  end
end
```

Key decisions:
- Uses `insert_conflict(target: :output_id)` for atomic contention handling
- If any input fails to lock, rolls back the entire transaction
- Returns a plain hash with `:id`, `:reference`, `:broadcast`
- The caller (engine or UTXOPool) decides what to do with the failure

**`promote_action(action_id:, outputs:)`** — Phase 4

The largest atomic operation — writes to 5 tables in one transaction.

```ruby
def promote_action(action_id:, outputs:)
  db.transaction do
    outputs.each do |out|
      output = Output.create(
        action_id: action_id,
        satoshis: out[:satoshis],
        vout: out[:vout],
        locking_script: out[:locking_script],
        derivation_prefix: out[:derivation_prefix],
        derivation_suffix: out[:derivation_suffix],
        sender_identity_key: out[:sender_identity_key]
      )

      Spendable.create(output_id: output.id)

      if out[:basket]
        basket_id = find_or_create_basket(name: out[:basket])
        OutputBasket.create(output_id: output.id, basket_id: basket_id)
      end

      if out[:description] || out[:custom_instructions] || out[:change]
        OutputDetail.create(
          output_id: output.id,
          description: out[:description],
          custom_instructions: out[:custom_instructions],
          change: out[:change] || false
        )
      end

      if out[:tags]&.any?
        tag_ids = find_or_create_tags(names: out[:tags])
        tag_ids.each { |tid| OutputTag.create(output_id: output.id, tag_id: tid) }
      end
    end
  end
end
```

#### Queries

**`find_action(id:, txid:, reference:)`**

Finds by whichever key is provided. Returns a hash with `derived_status` included.

```ruby
def find_action(id: nil, txid: nil, reference: nil)
  record = if id then Action[id]
           elsif txid then Action.first(txid: txid)
           elsif reference then Action.first(reference: reference)
           end
  return unless record
  action_to_hash(record)
end
```

**`query_actions(labels:, label_query_mode:, ...)`**

The `:any` vs `:all` distinction:
- `:any` — action has at least one of the specified labels (OR)
- `:all` — action has ALL of the specified labels (AND, via HAVING COUNT)

```ruby
def query_actions(labels:, label_query_mode: :any, limit: 10, offset: 0, **includes)
  label_ids = Label.active.where(label: labels).select_map(:id)
  return { total: 0, actions: [] } if label_ids.empty?

  base = Action.distinct
    .join(:action_labels, action_id: :id)
    .where(Sequel[:action_labels][:label_id] => label_ids)
    .where(Sequel[:action_labels][:deleted_at] => nil)

  if label_query_mode == :all
    base = base.group(Sequel[:actions][:id])
      .having { count(distinct(Sequel[:action_labels][:label_id])) >= label_ids.size }
  end

  total = base.count
  records = base.order(Sequel.desc(Sequel[:actions][:created_at]))
    .limit(limit).offset(offset)
    .select_all(:actions)
    .map { |a| action_to_hash(a, **includes) }

  { total: total, actions: records }
end
```

**`query_outputs(basket:, tags:, tag_query_mode:, ...)`**

Similar pattern — spendable outputs in a basket, optionally filtered by tags.

```ruby
def query_outputs(basket:, tags: nil, tag_query_mode: :any, limit: 10, offset: 0, **includes)
  base = Output.spendable.in_basket(basket)

  if tags&.any?
    tag_ids = Tag.active.where(tag: tags).select_map(:id)
    tag_ds = OutputTag.active.where(tag_id: tag_ids)
      .where(Sequel[:output_tags][:output_id] => Sequel[:outputs][:id])

    if tag_query_mode == :all
      base = base.where(
        tag_ds.group(Sequel[:output_tags][:output_id])
          .having { count(distinct(Sequel[:output_tags][:tag_id])) >= tag_ids.size }
          .select(1).exists
      )
    else
      base = base.where(tag_ds.select(1).exists)
    end
  end

  total = base.count
  records = base.order(Sequel.desc(:created_at))
    .limit(limit).offset(offset)
    .map { |o| output_to_hash(o, **includes) }

  { total: total, outputs: records }
end
```

#### UTXO Selection

**`find_spendable(satoshis:, basket:, exclude:)`**

The hot-path query. Enters through spendable, PK-joins to outputs, anti-joins on inputs.

```ruby
def find_spendable(satoshis:, basket: nil, exclude: [])
  ds = Output.spendable
  ds = ds.in_basket(basket) if basket
  ds = ds.exclude(Sequel[:outputs][:id] => exclude) if exclude.any?
  ds = ds.order(Sequel.desc(:satoshis))

  candidates = []
  total = 0
  ds.each do |output|
    candidates << output_candidate_hash(output)
    total += output.satoshis
    break if total >= satoshis
  end
  candidates
end
```

Stops scanning as soon as we have enough — doesn't load the entire UTXO set.

**`reap_stale_actions(threshold:)`**

```ruby
def reap_stale_actions(threshold:)
  cutoff = Time.now - threshold
  Action.where(Sequel.lit('txid IS NOT NULL'))
    .exclude(broadcast: 'none')
    .where { created_at < cutoff }
    .exclude(Action.where(
      Sequel[:outputs][:action_id] => Sequel[:actions][:id]
    ).select(1).exists)
    .delete
end
```

### Private helper methods

Two serialization helpers convert models to interface-boundary hashes:

- `action_to_hash(record, **includes)` — converts an Action to a hash, optionally including labels, inputs, outputs based on the `include_*` flags
- `output_to_hash(record, **includes)` — converts an Output to a hash, optionally including locking_script, tags, labels, custom_instructions

These ensure no Sequel::Model objects leak through the interface.
